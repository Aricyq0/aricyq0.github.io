本课主要讨论了多线程编程（Multi-threaded Programming）及错误处理（Error Handling），重点在于如何在 Swift 语言中使用异步（async）和等待（await）关键字来确保移动应用的 UI 始终保持响应性，同时正确地处理可能出现的错误。

1. 课程前言

  • 首先简要介绍了 Color 和 Image 相关的 Swift 结构体，并强调它们如何在 UI 中使用（Color vs UIColor，Image vs UIImage）。
  • 然后，正式进入本课主题：多线程编程（Multi-threaded Programming）。多线程编程的目标：确保 UI 不会被阻塞，即使应用程序正在执行耗时操作（如从网络加载数据）。
  • 另一个关键点是：错误处理，因为在多线程编程中，错误处理至关重要。

2. 颜色（Color）与图像（Image）

2.1 颜色的不同表示方式

Swift 中有多个用于表示颜色的类型：

  • Color：

    • 可用于 color-specifier： foregroundColor(Color.green) 等
    • 也可以用于 ShapeStyle：.fill(Color.blue)
    • 作为 View：Color.white。

  • UIColor：

    • 用于实际表示颜色（在不同色彩空间，如 RGB、HSB）。
    • 可以通过 Color(UIColor:) 转换为 Color。

  • CGColor：

    • Core Graphics 的底层颜色表示方式。
    • 主要用于底层绘图 API，很少直接使用。

2.2 图像的不同表示方式

类似于颜色，图像在 Swift 中也有不同的表示方式：

  • Image：

    • SwiftUI 结构体，用于在 UI 上显示图像。
    • 可以使用 systemName 加载 SF Symbols，也可以从 Assets 目录加载图片。

  • UIImage：

    • UIKit 中的类，可用于处理图像数据，如从 Data 加载图像。
    • 需要从 URL 加载图像时，必须先转成 UIImage，然后才能转换为 Image。

3. 多线程

3.1 为什么需要多线程？

  • UI 线程（主线程）需要保持流畅，不能被阻塞。
  • 阻塞 UI 的情况：
    • 网络请求（例如加载远程图片）。
    • 复杂的计算任务（如机器学习推理）。
    • IO 操作（如读取大文件）。
  • 解决方案：使用线程来并行处理任务。

3.2 线程基础

  • 现代操作系统支持多线程（Multi-threading）。
  • 大多数设备是多核（Multi-core），可真正并行执行多个线程。
  • 线程由系统管理，我们只需要指示何时在新线程上运行任务。

3.3 Task（任务）

Swift 提供了一些 API 来管理多线程任务：

  • 作用：在Swift中，使用Task可以异步执行耗时操作（如网络请求、复杂计算等），避免阻塞主线程。
  • 创建方式：
    • 常用方式：直接启动一个不跟踪的Task
  • 特性：
    • 任务会“立即或稍后”执行，但不会阻塞当前线程。
    • 可通过task.cancel()取消任务，或使用Task.sleep()、Task.yield()控制执行流程。

3.4 Actor（执行者）

  • 作用：解决多线程并发访问共享数据的冲突问题（如多个线程同时修改同一数组）。
  • 特性：
    • 引用类型（类似class），但不支持继承。
    • 同步机制：同一时间仅允许一个函数访问其数据（通过async函数挂起/恢复实现）。
    • 所有属性和函数默认受保护，需通过异步上下文（await）访问。

3.5 async与await

Async 放在可能被暂停的函数上。当我们使用 async 声明异步函数时，需要使用 await 关键词来调用。就像调用一个函数 throws 时，必须使用 try。

  • 规则：
    • **async**：标记可能挂起的函数（如网络请求）。
    • **await**：调用async函数时必须使用，表示此处可能挂起当前任务。
  • 关键点：
    • 函数中若包含await，自身必须标记为async。
    • 挂起期间，其他线程可能修改共享数据，恢复后需重新验证数据状态。

5.3 emojiArt 被修改时检查 background 状态

  • 检查 background 是否发生改变，如果是，则抓取背景图片（fetchBackgroundImage），并将通过 URLSession 从 url 抓取数据，转为 UIImage 类型。

  • 通过 URLSession 抓取数据是异步的，因此需要为 fetchUIImage 添加异步函数关键词 async，同时在 URLSession 前添加 await 关键词。另外，这个调用可能会报错，因此我们还需要在 URLSession 前添加 try ，同时在 fetchUIImage 添加 throws 关键词。

  • 如果抓取失败，则抛出错误 FetchError (使用 Enum 来创建)

  • fetchUIImage 也需为异步，需要添加 async 和 await，并处理背景返回错误

  • 对于 @Published 变量 emojiArt 中 fetchBackgroundImage() 是异步函数，但 emojiArt 不是异步函数，我们使用 Task 来解决报错 'async' call in a function that does not support concurrency
