The lectures for the Spring 2023 version of Stanford University’s course CS193p (Developing Applications for iOS using SwiftUI) were given in person but, unfortunately, were not video recorded. 0 课程概述

本讲主要围绕 SwiftUI 的导航（Navigation）、菜单（Menu）、表单（Form）及弹出窗口（Modal Views） 展开，特别关注 如何构建 Palette 编辑界面。其中涵盖了 contextMenu、NavigationStack、NavigationSplitView、Sheet、Popover 及 List 等 SwiftUI 组件的使用。

1. 课程引入

  • 本讲主要是一个 大型示例演示，重点在于 如何管理 Palette（调色板），类似于作业 A6 需要管理 Memorize 主题的方式。
  • 目标：创建一个 Palette 管理界面，用户可以：
    • 选择 Palette
    • 编辑 Palette（修改名称、添加/删除表情符号）
    • 通过菜单快速导航
    • 使用 NavigationStack 进行层级导航

3.3.3 修正字体传递问题

  • 因为 sheet 继承了父视图的字体，使得弹出窗口的字体过大。
  • 解决方案：显式设置 font(nil)，取消继承字体大小，恢复默认大小

但此时它们都太小了，我们想要它在被删除时或者被点击时能够放大，字体大小与 palette 编辑器的图标大小一样，因此为它单独设置字体。

3.3.4 Form 绑定 palette.name 和 palette.emojis，同步编辑

  • addEmojis 绑定：当添加 Emoji 时，palette.emojis 应同步添加，并支持 remove 操作。

3.8 实现侧边栏多窗格（NavigationSplitView）

3.8.1 需求

  • iPad 需要支持多窗口布局，即左侧是 Palette 列表，右侧是编辑界面。
  • 使用 NavigationSplitView 进行布局。

3.8.2 代码实现

  • 新建一个 swiftUI 文件 PaletteManger 管理所有的调色板。使用 NavigationSplitView 替代 NavigationStack 进行布局。

  • 将 Emoji_ArtApp.swift 中的 EmojiArtDocumentView 改为 PaletteManager

  • 创建一个 swiftUI 文件 PaletteStoreView，将 List 中的内容替换为 PaletteStoreView 详见课程附件代码。

  • 使用 List 布局 stores，PaletteStore 需符合 Identifiable 协议，如果不想使用这种方式，我们可以创建两个具有相同名字的 PaletteStore，将它们都写入相同的默认位置。

  • 将所有的商店放置在左侧窗格，将该商店中的调色板放在右侧窗格。

  • 取消预览

  • 在 EmojiArtApp 中创建多个 store

  • PaletteManager 中 List 绑定 selectedStore，同时声明为 @State。此时要求 PaletteStore 符合 Hashable 以及 Equatable 协议。

  • 为每个 store 添加 tag，用于突出被选择的项。

  • 添加 detail，即第二个窗格的内容（右侧窗格）：EditablePaletteList，因为我们需要传递不同的 store 供 EditablePaletteList 直接使用，它需要接受动态选择的 store，而 @EnvironmentObject 无法动态传递不同的 store，因此我们需要将 EditablePaletteList 中的 store 由注入模式修改为非注入模式，即 @EnvironmentObject 修改为 @ObservedObject。

  • 希望呈现第三个面板，在 PaletteManager 中添加 content，即将 paletteList 移入当前面板，此时我们希望在第二个面板中选择对应的 Palette 时，相关 Palette 的信息会呈现在第三个面板中，但是由于 EditablePaletteList 使用的是 NavigationStack，NavigationDestination 属性的设置仍然会使得相关信息在第二个面板中呈现(如下所示)。（当你需要额外构建第三个窗格时适用，此时不适用）
